# YouTube + Dailymotion 矩阵底表说明(供 mapleBI / IT)

## 数据链路(重要)
YouTube Data API(googleapis.com)**在中国大陆服务器访问不到**,所以采集必须在海外执行。当前链路:

```
GitHub Actions(海外,每天北京 9:23 / 12:30 兜底)
  → 调 YouTube Data API + Dailymotion API 采集
  → 生成底表 base_tables/*.csv
  → 提交回 GitHub 仓库 reelshort-yt-dm-dashboard
  → IT 拉取底表 → 灌入 mapleBI 数仓 → 搭看板
```
IT 只需定时拉取这 4 个 CSV 灌库即可,**不需要自己接 YouTube API**(国内也接不了)。

## 数据口径(先知道)
- 全部为**公开数据**(无收益/无私有分析)。播放/订阅来自平台公开接口,**比平台后台(Studio)通常低约 15–25%**,适合趋势与横向对比,不作对账口径。
- DM 频道总播放含已下架视频历史;视频级明细只含当前公开视频。
- 时间序列底表(*_daily)是**每日快照累积**,历史无法回填,从首次采集日起逐日生长。

## 四张底表

### 1. dim_channel.csv —— 频道维度表
| 列 | 说明 |
|---|---|
| platform | YouTube / Dailymotion |
| channel_id | 频道ID(YT 为 UC…,DM 为 x… 短码)。**主键** |
| channel_name | 频道名 |
| operator | 负责人(YT 来自 YouTube账号.xlsx 按 channel_id;DM 来自 owner.xlsx 按频道名) |

### 2. channel_daily.csv —— 频道级日快照(事实表,时间序列)
粒度:一行 = 一个频道一天。
| 列 | 说明 |
|---|---|
| snapshot_date | 快照日期 YYYY-MM-DD |
| platform / operator / channel_id / channel_name | 维度(可与 dim_channel 关联) |
| subscribers | 订阅/粉丝数(累计值) |
| total_views | 频道总播放(累计值,含已下架视频历史) |

### 3. video_latest.csv —— 视频级最新快照(事实表,当前状态)
粒度:一行 = 一条当前公开视频(最新值)。用于 Top 视频、人效比、互动率等。
| 列 | 说明 |
|---|---|
| platform / operator / channel_id / channel_name | 维度 |
| video_id | 视频ID。**主键** |
| video_title | 标题 |
| published_date | 发布日期 |
| duration_sec | 时长(秒) |
| views / likes / comments | 播放 / 点赞 / 评论(累计;DM 点赞/评论可能为空) |

### 4. video_daily.csv —— 视频级日快照(事实表,时间序列)
粒度:一行 = 一条视频一天。用于单视频/频道的**按天播放趋势**(当日新增 = 相邻两天 views 之差)。
| 列 | 说明 |
|---|---|
| snapshot_date | 快照日期 |
| platform / operator / channel_id | 维度 |
| video_id | 视频ID(关联 video_latest 取标题等) |
| views | 当日累计播放 |

## 关联关系
- `*_daily` / `video_latest` 都可用 `channel_id` 关联 `dim_channel`,用 `video_id` 互相关联。
- 编码 UTF-8;CSV 含表头。

## 更新频率
每天北京时间约 9:23(12:30 兜底)刷新一次。
